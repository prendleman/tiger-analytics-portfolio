# Security

This repository is a **portfolio project**. It uses synthetic data only (no real PHI or production credentials). It is not intended for production deployment.

- **Data**: All healthcare data is generated by `generate_healthcare_data.py` (Faker + pandas). No real patient or claim data.
- **Secrets**: Do not commit API keys, tokens, or credentials. The repo uses no secrets for normal operation.
- **Dependencies**: CI runs tests and dependency checks. Keep `requirements.txt` (and any other dependency files) updated.

If you believe you have found a security issue, please open a **private** GitHub Security Advisory for this repository, or contact the repository owner. Do not open a public issue for security vulnerabilities.
