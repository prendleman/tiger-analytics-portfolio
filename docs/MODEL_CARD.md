# Model card: 30-day readmission (binary classification)

## Overview

- **Task**: Binary classification — will this encounter lead to a 30-day readmission?
- **Model**: Random Forest (scikit-learn); optional hyperparameter tuning via GridSearchCV (see notebook).
- **Data**: Synthetic healthcare encounters with demographics, LOS, diagnosis count, lab summary, encounter type.

## Intended use

- **Purpose**: Portfolio demonstration of end-to-end ML (feature engineering, train/test, evaluation, tuning). Not for clinical or operational use.
- **Users**: Recruiters, hiring managers, technical interviewers reviewing the repo.
- **Out-of-scope**: Real PHI, production deployment, clinical decision support.

## Training data

- **Source**: `data/raw/` (generated by `generate_healthcare_data.py`). Synthetic; no real patients.
- **Target**: `readmissions.is_30_day` joined to encounters.
- **Features**: Length of stay, age at admission, diagnosis count per encounter, lab mean/count, encounter type (dummies).
- **Train/test**: 75/25 stratified split; fixed random_state=42.

## Evaluation

- **Metrics**: Precision, recall, F1, ROC-AUC, confusion matrix (see notebook and script output).
- **Limitations**: Class imbalance (few readmissions); no temporal validation or external holdout.

## Ethical considerations and limitations

- Data is synthetic; model has not been validated on real outcomes.
- Do not use for any clinical, reimbursement, or policy decisions.
- Bias and fairness were not formally assessed.

## Reproducibility

- `python src/python/readmission_model.py` (from repo root, with data in `data/raw/`).
- Notebook: `notebooks/python/readmission_prediction.ipynb` (includes GridSearchCV).
